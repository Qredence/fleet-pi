# Fleet Pi Self-Improvement Policy

Self-improvement is allowed, but it should stay explicit and reviewable.

Every self-improvement should answer four questions:

1. What failure, friction, or recurring pattern triggered this?
2. What changed?
3. How will future agents benefit?
4. How can the change be evaluated?

Good self-improvements turn repeated human judgment into reusable guidance,
checks, or small workflow refinements. They should appear as normal repository
diffs, not hidden mutations in transient state.
