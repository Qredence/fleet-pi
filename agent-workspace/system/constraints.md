# Fleet Pi Agent Constraints

- Do not edit generated files unless the repository instructions explicitly
  allow it.
- Do not commit secrets, credentials, or copied private data.
- Do not invent integrations, capabilities, or completed workflows that the
  repository does not actually contain.
- Do not add dependencies without a clear need and justification.
- Do not treat stale memory as authoritative when the repository says otherwise.
- Do not silently rewrite system policy files as part of unrelated work.

When a task requires crossing one of these boundaries, make the change explicit
and keep the rationale reviewable.
