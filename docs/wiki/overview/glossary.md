# Glossary

This glossary uses qualified terms when Fleet Pi has more than one concept with the same everyday name. Executable types and accepted ADRs are the authority for current behavior.

| Term                         | Definition                                                                                                                                                                                                                                                             |
| ---------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Fleet Pi**                 | The browser-based product that hosts Pi coding-agent sessions, project tools, durable adaptive state, and optional isolated sandboxes.                                                                                                                                 |
| **Pi**                       | Qredence's AI coding agent (`@earendil-works/pi-coding-agent`), which manages model sessions, built-in tools, and Pi resources.                                                                                                                                        |
| **canonical adaptive state** | Durable, reviewable state under `agent-workspace/`, including memory, plans, skills, evals, artifacts, and workspace-installed Pi resources. Indexes, caches, databases, and live runtimes are not canonical adaptive state.                                           |
| **canonical transcript**     | The persisted Pi JSONL session file containing messages, tool calls, and results. The Neon mirror is derived data and may be an authenticated recovery source after a Vercel cold start, but it is not a second canonical transcript.                                  |
| **Pi session**               | A persisted JSONL conversation identified by a session file and session ID. It can contain multiple chat runs.                                                                                                                                                         |
| **live runtime**             | The transient in-memory `AgentSessionRuntime` serving a Pi session. It is retained for a short TTL and recreated from the canonical transcript when needed.                                                                                                            |
| **chat run**                 | One streamed execution of a user turn within a Pi session. A run is not a session.                                                                                                                                                                                     |
| **Agent mode**               | Chat mode with normal coding tools (`read`, `write`, `edit`, `bash`) plus the approved external tools configured by the runtime.                                                                                                                                       |
| **Plan mode**                | Read-only chat mode for inspection and planning. It permits inspection tools and questionnaire flows while blocking file mutation and other side effects.                                                                                                              |
| **Harness mode**             | Constrained workspace-governance and resource-management mode. It permits governed workspace updates and `resource_install` while blocking general `edit`/`write` mutation tools; it is not Agent mode plus `questionnaire`.                                           |
| **agent workspace**          | The durable `agent-workspace/` adaptive-state directory. Use **project root**, **sandbox workspace root**, or **Workspace panel** when referring to those different concepts.                                                                                          |
| **plan artifact**            | A durable plan file under `agent-workspace/plans/`. This is distinct from Plan mode, transient chat plan state, a plan action (`execute` or `refine`), and plan execution.                                                                                             |
| **Pi resource**              | A skill, prompt, extension, theme, or package understood by Pi. A workspace-installed resource lives under `agent-workspace/pi/`; executable resources may be staged, active, or reload-required.                                                                      |
| **NDJSON**                   | Newline-delimited JSON, the streaming format used by `/api/chat`. Each line is a `ChatStreamEvent`: `start`, `delta`, `tool`, `plan`, `state`, `queue`, `thinking`, `compaction`, `retry`, `done`, or `error`. `compaction` and `retry` have start/end phase variants. |
| **Pi extension**             | A TypeScript module under `.pi/extensions/` or a workspace-installed extension under `agent-workspace/pi/extensions/` that registers tools or patches Pi behavior.                                                                                                     |
| **Pi skill**                 | A Markdown procedure under `.pi/skills/` or `agent-workspace/skills/` that Pi can activate for a workflow.                                                                                                                                                             |
| **Daytona sandbox**          | An isolated container managed by Daytona. When enabled, Pi tool operations run against the user's sandbox workspace root rather than the local project filesystem.                                                                                                     |
| **OpenUI**                   | Fleet Pi's structured generative-UI format and renderer, backed by the OpenUI protocol package and hax-design components.                                                                                                                                              |
| **Better Auth**              | The authentication system for user accounts and auth sessions, backed by SQLite locally or Neon Postgres when configured.                                                                                                                                              |
| **Auth session**             | A Better Auth authentication session. Do not call it a Pi session or live runtime.                                                                                                                                                                                     |
| **session mirror**           | Optional Neon Postgres derived data for Pi sessions, runs, events, tool executions, and file mutations. Pi JSONL remains the canonical transcript.                                                                                                                     |
| **Pi session mirror row**    | A `pi_sessions` record representing a mirrored Pi session. It is not the Pi session file itself and is owner-scoped on Vercel.                                                                                                                                         |
| **recovery source**          | The authenticated Neon mirror used to reconstruct an ephemeral Pi JSONL copy after a Vercel cold start. Recovery does not change canonical transcript ownership.                                                                                                       |
| **workspace index**          | A regenerable projection over `agent-workspace/` files used for query and search. It is not canonical adaptive state.                                                                                                                                                  |
| **provenance**               | Metadata tracking which chat run or tool execution mutated a canonical filesystem path. It is separate from transcript ownership and mirror deletion.                                                                                                                  |
| **tool allowlist**           | The tools Pi may call in a given chat mode, enforced by the mode extension and server runtime.                                                                                                                                                                         |
| **deployment trust zone**    | `local`, `vercel-production`, or `vercel-preview`; it drives environment readiness and authentication policy.                                                                                                                                                          |
| **primary provider**         | An OpenAI-compatible provider through Pi's `openai-chat-completions` integration, currently configured with model `deepseek-v4-flash-free`. Google Gemini, Amazon Bedrock, and other providers remain optional integrations.                                           |
| **FLEET_PI_REPO_ROOT**       | Environment variable that overrides the project root Pi operates in; it defaults to `process.cwd()`.                                                                                                                                                                   |
| **Autocontext**              | External npm package (`npm:pi-autocontext`) that provides workspace context-management tools.                                                                                                                                                                          |
